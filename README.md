# ЦОД категории III на 120 стоек — сайт проекта

Концепция новой нетарифной услуги для филиала ПАО «Россети Юг» — «Кубаньэнерго»: размещение вычислительного и телекоммуникационного оборудования в инженерном комплексе на **120 стоек** с расчётной полезной нагрузкой **840 кВт**.

---

## Структура проекта

```
.
├── index.html              ← главная: hero, KPI, разделы
├── glossary.html           ← термины и сокращения
├── roadmap.html            ← дорожная карта реализации
├── requirements.html       ← требования к объекту
├── substations.html        ← площадка и инфраструктура
├── pricing.html            ← тарифы и пакеты (объединено: pricing + tariffs)
├── business-plan.html      ← бизнес-план
├── investor.html           ← инвестор и финансирование
├── risks.html              ← риски и управление
├── contract.html           ← правовая конструкция
├── templates.html          ← пакет документов
├── assets/
│   ├── styles.css          ← общая таблица стилей (один файл на всех)
│   └── app.js              ← общий JS (бургер-меню, FAB, восстановление прокрутки)
└── README.md
```

11 страниц, **одна** общая таблица стилей и **один** общий скрипт. Никакого дублирования CSS внутри HTML — изменение цвета или шрифта правится в одном месте.

## Что изменилось при переписывании

**Структурно**

- Из 14 исходных страниц убраны 3 дубликата (`businessplan.html` объединён с `business-plan.html` — взята более детальная версия; `tariffs.html` объединён с `pricing.html`; `documents.html` объединён с `templates.html`).
- В основное меню добавлены страницы `investor.html` и `risks.html` — раньше они существовали, но не были связаны с главной.
- Вся разметка переведена на единый шаблон: общая шапка-навигация, общий футер, общий фон.

**Технически**

- ~6 000 строк дублированного CSS (по ~430 в каждой странице) → один файл `assets/styles.css` (~800 строк).
- Убраны inline `<style>` блоки на каждой странице.
- Sticky-навигация сверху на всех страницах с подсветкой активного раздела и адаптивным бургер-меню для мобильных.
- Между разделами — нижний пейджер «← Предыдущий / Следующий →».
- На главной — восстановление позиции прокрутки при возврате с внутренней страницы (через `sessionStorage`).

**Дизайн**

- Шрифты: **Unbounded** (заголовки, геометрический гротеск с поддержкой кириллицы) + **Onest** (основной текст). Без Inter/Roboto/системных.
- Палитра уточнена: `#050a1f` (фон) → `#0a1230` → `#1a237e` → `#3d4ec5` → `#00d4e8` (акцент). Чуть глубже и контрастнее исходной.
- Фон с дрейфующими радиальными градиентами + ненавязчивая координатная сетка.
- KPI-блоки со встроенными числами (120 стоек / 840 кВт / 216–259,2 млн ₽ / 0,78–0,90 млрд ₽).
- Карточки разделов с эффектом «свечения от курсора».

**Контент**

- Сохранён весь исходный текст: каждый параграф, каждая цифра, каждый список — на месте.
- Тарифная страница объединена корректно: принципы (из `pricing`) → конкретные ставки 150/170/180 тыс. (из `tariffs`) → конструктор опций → таблица базового пакета → ориентиры выручки → допуслуги.

## Как развернуть на GitHub Pages

### Если репозиторий называется `Collocation` (как сейчас)

1. **Залить файлы в репозиторий.** Все HTML — в корень, папка `assets/` рядом.

   ```bash
   # из локальной папки с распакованным архивом
   git init
   git add .
   git commit -m "Полный rewrite сайта: общий design system, 11 страниц"
   git branch -M main
   git remote add origin https://github.com/<ваш-логин>/Collocation.git
   git push -u origin main
   ```

   Если репозиторий уже существует — просто замените файлы и сделайте `git push`.

2. **Включить GitHub Pages.** На странице репозитория → **Settings** → **Pages**:
   - **Source:** `Deploy from a branch`
   - **Branch:** `main`, папка `/ (root)`
   - **Save**

3. **Дождаться сборки** (~30–60 секунд). GitHub покажет URL вида:
   ```
   https://<ваш-логин>.github.io/Collocation/
   ```

### Свой домен (опционально)

Если будет домен, например `dc.example.ru`:

1. Добавьте файл `CNAME` в корень с содержимым `dc.example.ru`.
2. В DNS пропишите CNAME-запись на `<ваш-логин>.github.io`.
3. В **Settings → Pages → Custom domain** введите `dc.example.ru` и дождитесь выдачи SSL-сертификата.

### Локальный просмотр перед публикацией

Просто откройте `index.html` в браузере **через локальный сервер** (не файловый протокол `file://`, иначе шрифты Google Fonts могут не подгрузиться):

```bash
# в папке проекта
python3 -m http.server 8000
# открыть http://localhost:8000
```

## Как редактировать

**Изменить цвет или шрифт по всему сайту** — `assets/styles.css`, секция `:root { ... }` в начале файла. Все цвета и размеры идут через CSS-переменные.

**Изменить пункт меню** — отредактируйте блок `<div class="nav-links">` сразу на всех страницах (поиск-замена в редакторе) или поправьте список в `assets/app.js` (там же активная подсветка).

**Добавить раздел** — создайте новый `имя.html` по образцу любого существующего (например, `glossary.html`), добавьте ссылку в `nav-links` на всех страницах и карточку на главной.

**Изменить тарифы / цифры** — открыть нужную страницу и поправить текст. Все числа в обычном HTML, никакой шаблонизации.
